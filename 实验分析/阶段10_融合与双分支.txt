阶段10 融合与双分支实验分析

本阶段的目标是直接验证上一阶段提出的两个后续方向：第一，传统统计基线与窗口序列模型之间是否存在可直接利用的互补性；第二，如果简单输出融合不足以带来收益，那么把两类信息放进同一个轻量双分支模型中进行联合建模，是否能够在保持 coarse signal 稳定性的同时，引入动态时序信息并获得进一步提升。为此，本阶段围绕两条路线展开：一条是概率级融合实验，另一条是特征级双分支实验。所有实验仍然沿用前面一致的评估协议，即训练集上的被试级 5 折交叉验证以及固定 hold-out 被试集评估。

首先，在概率融合实验中，我们分别测试了两组组合：baseline_logreg_meanstd + stage2_window_mean，以及 baseline_logreg_meanstd + window_mlp_gru。具体做法是，在每个 GroupKFold 折内分别训练传统基线与窗口模型，并在验证折上输出正类概率，再对两者做加权平均，权重从 0 到 1 按 0.1 递增搜索，最终以 out-of-fold 准确率最高的权重作为该组合的最优融合权重。实验结果表明，这两组概率融合均未带来增益。对于 baseline + stage2_window_mean，最优权重落在 baseline_weight = 0.0，即完全退化为只使用 stage2_window_mean，本身的 out-of-fold 准确率约为 0.669，hold-out 准确率约为 0.677。对于 baseline + window_mlp_gru，最优权重同样落在 baseline_weight = 0.0，也就是完全退化为只使用 window_mlp_gru，对应 out-of-fold 准确率约为 0.698，hold-out 准确率约为 0.677。换句话说，一旦把 baseline 的概率真正加进去，结果并没有超过单独窗口模型，反而通常会下降。

这一现象说明，当前两条模型线虽然在表示层级上不同，但它们的输出概率并没有形成足够稳定的“互补误差结构”。如果两者只是以不同方式重复利用了部分相近信息，那么简单的后验加权平均不会自然提升性能；相反，概率融合还可能把 baseline 的稳定性和序列模型的偏置同时混在一起，导致整体结果不升反降。从本阶段结果看，当前“简单集成”并不能回答“复杂模型学到的信息能否在 baseline 约束下转化为真实收益”这个问题，至少在输出层线性融合这一层面，答案是否定的。

随后，我们进一步转向轻量双分支模型。双分支模型中，分支 A 输入 trial 级统计特征，也就是当前最优 baseline 使用的 mean+std + classic5 DE 特征；分支 B 输入窗口级序列特征，也就是 stage2/window_mlp_gru 这条线使用的 2 秒窗口张量表示。两个分支分别编码后，在中间进行融合，再由统一分类头输出最终类别。本阶段共测试了 4 个版本：mean + concat、mean + gated、gru + concat 和 gru + gated。其中，mean 版本对应较轻的时间均值聚合，gru 版本对应更强的时序动态建模；concat 表示直接拼接两个分支表征，gated 表示通过门控在两个分支之间学习自适应权重。

双分支实验结果明显比简单概率融合更有价值。首先，mean 系列模型虽然比单独 stage2_window_mean 更强，但仍未超过 baseline。dual_branch_mean_concat 的交叉验证平均准确率约为 0.696，hold-out 约为 0.708；dual_branch_mean_gated 的交叉验证平均准确率约为 0.694，hold-out 约为 0.719。说明在较弱的时间聚合方式下，即使加入统计分支，仍然不足以稳定超越当前最优传统基线。不过，mean_gated 在 hold-out 上已经接近 baseline，并且正常组与抑郁组准确率均约为 0.719，组间差几乎为 0，说明门控机制在这一版本中确实起到了平衡两类信息源的作用。

更值得关注的是 gru 系列。dual_branch_gru_gated 的交叉验证平均准确率约为 0.702，hold-out 准确率约为 0.729，与 baseline 基本持平；其 hold-out 上抑郁组约为 0.688，正常组约为 0.750，组间差约为 0.063，虽然略高于 baseline，但仍处于可接受范围。相比之下，dual_branch_gru_concat 的结果最强：交叉验证平均准确率约为 0.704，hold-out 准确率约为 0.740，首次在当前轻量深度主线上超过 baseline 的约 0.729。这说明，trial 级统计表征与窗口级时序表征之间并非完全没有协同空间；只是这种协同关系没有办法通过简单的输出平均直接体现，而需要在特征层由统一模型共同学习。

不过，dual_branch_gru_concat 同时也暴露了一个重要问题：性能提升伴随着明显的组间偏差扩大。该模型在 hold-out 上抑郁组准确率约为 0.875，而正常组仅约为 0.672，组间差约为 0.203，远高于 baseline 的约 0.031。这意味着，虽然双分支联合建模的确把动态时序信息转化成了总体准确率增益，但目前最优版本的提升方式并不完全“干净”，它仍然较大程度地偏向了某一群体。换句话说，本阶段首次证明了“统计分支 + 时序分支”这条路是有潜力的，但也同时说明，如果不加任何约束，模型会优先放大那些能提分但不一定足够公平和稳健的模式。

从本阶段结果可以得到三个关键结论。第一，简单概率融合并不能带来提升，说明当前 baseline 与序列模型的互补性不能只靠输出层加权来利用。第二，轻量双分支模型是有效方向，尤其是 GRU 序列分支与统计分支的特征级联合，已经能够在 hold-out 上首次超过当前 baseline。第三，这种提升目前仍然伴随较大的群体偏置，因此“双分支”虽然已经可以作为新的方法主线，但下一步不能只继续追求更高分数，而应当同时考虑如何把组间 gap 正式纳入模型选择标准。

综合来看，本阶段基本验证了上下文中给出的路线图：短期内，单纯融合不是最优解；真正有前景的是“trial 级统计先验 + 轻量时序分支”的双分支方法。当前最值得继续推进的模型，不再是简单概率融合，而是 dual_branch_gru 系列。其中，dual_branch_gru_concat 适合作为“冲击更高分数”的版本，因为它已经在 hold-out 上取得当前最佳结果；dual_branch_gru_gated 则更适合作为“更稳健、更平衡”的版本，因为它在保持 baseline 级别总体性能的同时，组间差明显小于 gru_concat。后续如果继续推进，本阶段最自然的下一步不是回到图主干或更重的时空模型，而是在双分支框架内进一步研究：如何通过门控设计、损失约束或显式组间 gap 指标，把当前已经观察到的双分支增益转化为更稳定、更可解释的提升。
