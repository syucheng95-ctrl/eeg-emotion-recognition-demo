阶段8 预处理消融实验分析

本阶段针对前处理是否限制了模型性能这一问题，设计了一个小规模但结论明确的消融实验。实验对象固定为两条当前最有代表性的主线：一条是传统强基线（2s窗长、2s步长、classic5频带、mean+std聚合、Logistic Regression），另一条是当前表现最好的非图序列模型（stage2_window_mean，即窗口级 MLP 编码 + 时间均值聚合）。评估协议保持不变，仍采用被试级 5 折交叉验证和固定 hold-out 被试集。

本轮共比较四种前处理设置：原始版本、仅加入通道内 z-score、仅加入 local baseline correction、以及同时加入 z-score 和 baseline correction。其中，通道内 z-score 指对每个 trial 的每个通道单独做零均值单位方差标准化；local baseline correction 指在每个 2 秒窗口内，以前 0.5 秒为局部 baseline，按通道减去该 baseline 均值。考虑到原始版本结果已经在前一轮实验中完成，本阶段仅补跑后三组，并与已有原始结果进行统一比较。

实验结果显示，通道内 z-score 对传统 DE 基线产生了明显负面影响。原始传统基线在 hold-out 上的准确率约为 0.729，且抑郁组与正常组性能差距仅约 0.031；加入 z-score 后，hold-out 准确率下降到约 0.583，组间差扩大到约 0.172。这说明 trial 内通道标准化会削弱 DE 特征所依赖的幅值/能量差异，而这些差异在当前任务中恰好包含了有效判别信息，因此不适合直接并入当前传统主线。进一步地，加入 z-score + baseline correction 的结果与仅加 z-score 基本一致，也说明 z-score 是这一负面变化的主要来源。

相比之下，local baseline correction 对传统基线几乎没有影响。原始版本和加入 baseline correction 后的传统基线在交叉验证、hold-out 准确率和组间差上几乎一致。这一现象与 DE 特征的定义是吻合的：DE 本质上依赖带通后信号的方差，而单纯做窗口内通道均值平移不会显著改变方差，因此不会明显改变最终的频带能量统计。这说明，在当前比赛数据中，使用“窗口内前 0.5 秒局部去均值”这种保守 baseline correction，并不能为传统 DE 主线带来额外收益。

对序列模型而言，现象有所不同。原始 stage2_window_mean 的 hold-out 准确率约为 0.729，但对抑郁组和正常组的性能差异很大，抑郁组约为 0.875，正常组约为 0.656，组间差达到约 0.219，存在明显的人群偏置。加入 z-score 后，序列模型的 hold-out 准确率下降到约 0.677，但组间差显著缩小到约 0.016；加入 baseline correction 后，hold-out 准确率约为 0.708，组间差同样缩小到约 0.016；两者同时加入时，结果与仅加 z-score 基本一致。这说明，无论是通道内 z-score 还是局部 baseline correction，都会在一定程度上削弱序列模型对群体差异的利用，使模型在正常组和抑郁组之间更均衡，但暂时没有转化成更高的总体泛化性能。

综合来看，这轮实验得出四点结论。第一，通道内 z-score 不适合作为当前 DE 主线的默认前处理，因为它会破坏对传统统计特征有用的幅值信息。第二，local baseline correction 对当前传统 DE 基线几乎无效，说明简单的局部去均值不足以改变最终频带统计。第三，这两种前处理都能明显减弱序列模型的组间偏置，表明复杂模型确实在部分依赖群体相关模式。第四，单靠前处理还不足以同时解决“降低群体偏置”和“提高总体准确率”这两个目标，因此当前更合理的方向仍然是保留原始传统基线和原始序列模型，再进一步探索两者的融合，而不是直接把 z-score 或 baseline correction 并入默认主线。
