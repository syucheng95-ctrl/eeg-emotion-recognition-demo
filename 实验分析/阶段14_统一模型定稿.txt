阶段14 统一模型定稿

阶段14 的任务不是再把“强化统计主干”“更强交互”“群体约束”三条线继续并排推进，而是把它们收束成一个结构上统一、逻辑上自洽、结论上可落地的正式方法。基于阶段13已有结果，当前已经足够明确地给出一个“有机结合”的模型版本，而不必再把三条方向各自独立地重复扫一轮。原因在于，阶段13 实际上已经分别回答了三个关键子问题：第一，统计主干的增强方式不是无上限扩容，而是适度变宽，其中 stat_hidden=96、stat_num_layers=2 是当前最合理的甜点；第二，双分支交互中最有效的不是更自由的仿射调制，而是由统计分支单向调节时序分支的轻量 seq_gate；第三，群体约束中当前最稳的不是强对抗去群体化，而是小权重的轻度多任务监督。换句话说，三条线各自的第一轮筛选已经做完，阶段14 应做的是“组合定型”，而不是“重新散开”。

因此，当前可以明确给出的统一模型是：以强化统计分支作为主干，以 GRU 时序分支作为动态补充，以 seq_gate 作为统计到时序的条件交互，以 gated fusion 作为两分支融合机制，并在共享表示上加入轻量群体多任务监督。若写成一句完整的方法表达，就是：更强统计主干 + 统计调节时序 + gated 融合 + 轻群体监督。

这个统一模型的结构逻辑如下。首先，输入仍由两部分组成：一部分是 trial 级统计特征，即当前最稳的 mean+std + classic5 特征；另一部分是窗口级序列特征，即现有 window/graph cache 所对应的 trial 内时序张量。统计分支不再只是一个“陪衬支路”，而是整个模型的稳定锚点，因此其 MLP 宽度提升到 96，层数保持 2 层，不继续盲目加深。时序分支继续采用 2 层 GRU，对 trial 内窗口表示做轻量动态建模。随后，不是让统计分支和时序分支彼此独立地编码完再直接拼接，而是先利用统计分支输出的 stat_embedding 生成一个逐维门控向量，对 seq_embedding 做条件调节，即 seq_gate。这样做的含义不是让统计分支替代时序分支，而是让更稳定的 coarse-level 先验先去筛选和约束更易波动的动态表示。完成这一条件调节后，再通过 vector gated fusion 将 stat_embedding 与被调节后的 seq_embedding 融合，得到共享表示。最后，在共享表示上接两个头：主头负责情绪分类，辅头负责正常/抑郁群体分类；训练时使用 emotion loss + 小权重 group loss 的轻度多任务目标。

如果把这套方法和阶段13中的三条结论逐一对应，可以更清楚地看到它为什么是“有机结合”而不是“简单叠加”。强化统计主干解决的是“谁来当主干”的问题，答案是 trial 级统计表示；seq_gate 解决的是“统计主干如何与动态信息协同”的问题，答案不是末端平权，而是由统计表示单向调节时序表示；轻量群体监督解决的是“这种协同表示如何避免过度滑向群体差异”的问题，答案不是直接暴力去群体化，而是通过低权重监督把群体因素显式纳入并加以监控。三者放在一起，职责是清晰分工的：统计主干负责稳，交互模块负责把稳健先验注入动态表示，群体监督负责约束共享表示的偏置行为。这三者之间不是冲突关系，而是层次上自然衔接的关系。

基于现有结果，当前最应该作为统一模型默认配置的版本为：

stat_hidden_channels = 96
stat_num_layers = 2
temporal_type = gru
gru_layers = 2
fusion_type = gated
gate_mode = vector
interaction_mode = seq_gate
group_loss_weight = 0.02
dropout = 0.3
standardize_sequence_input = True

之所以把 group_loss_weight 暂定为 0.02，而不是更大的 0.05 或 0.1，是因为现有证据已经表明“轻约束”比“强约束”更符合当前任务特点。对抗版虽然在某些点上能冲高 hold-out，但组间 gap 放大明显；多任务版中更大的监督权重也没有稳定带来更优折中。相比之下，小权重 group loss 更适合作为“结构化约束”，而不是把群体目标推成训练主导项。因此，0.02 在当前阶段最适合作为统一模型的默认设置，它代表的是“让模型知道群体因素存在，并在共享表示中适度建模它”，而不是“让模型主要为群体区分服务”。

需要进一步说明的是，这个统一模型实际上已经在阶段13部分被间接验证过。interaction_seq_gate 的实验底座本身就已经包含强化统计主干和轻多任务监督，也就是说，“强化统计主干 + seq_gate + 轻群体监督”这件事并不是一个完全未经验证的新猜想，而是已有实验结果自然收束出的结构性结论。之所以阶段14仍然需要被命名为单独阶段，不是因为要重新从零证明这件事，而是因为要把这一组合从“几个独立实验结果的交集”正式上升为“项目的统一方法定义”。

因此，阶段14结束后，项目应明确区分两个层次的版本。

第一，统一正式版，也就是当前最推荐对外叙述、对内继续打磨的方法主体：
强化统计主干 + seq_gate 条件交互 + gated 融合 + 轻群体多任务监督。

第二，稳健保底版，也就是在强调泛化平衡性时可退回的版本：
强化统计主干 + 无额外交互 + gated 融合 + 轻群体多任务监督。

这两个版本的关系不是谁完全否定谁，而是一个强调“统一后的正式方法表达”，一个强调“组间平衡性最稳的保守版本”。从论文/答辩叙事上看，正式版更能体现方法创新，因为它真正把三条线收成了一个整体；从结果风险控制上看，保底版仍然有价值，因为它的 gap 更低，更适合作为稳定参照。换句话说，阶段14 的核心成果不是再找一个偶然高点，而是把“稳健主线”和“统一增强版”这两个角色彻底分清，并明确项目最终的方法学中心已经不再是图主干或简单集成，而是围绕统计主干、条件交互和轻群体约束构造的统一双分支模型。

最后，若只要求给出一个明确答案，而不是保留多个备选，那么当前阶段的明确统一模型可以直接定为：

更强统计主干（96x2） + 2层 GRU 时序分支 + seq_gate 单向条件交互 + vector gated fusion + 0.02 轻群体多任务监督。

这就是当前最符合已有全部实验结论、也最能体现“后三个方向有机结合”的模型。

为了避免这一结论仅仅建立在单个 seed 的偶然结果上，阶段14进一步对两条最有竞争力的联合配置做了复验比较。第一条是统一增强版候选：seq_gate + group_loss_weight=0.02；第二条是本轮补充扫描中意外冲高的对照候选：interaction=none + group_loss_weight=0.1。复验使用 seed 42、52、62 三次独立训练。结果显示，seq_gate + 0.02 三次 hold-out 情绪准确率分别为 0.7812、0.7500、0.7708，平均约为 0.7674，标准差约为 0.0130，平均 group gap 约为 0.0677。相比之下，none + 0.1 三次 hold-out 情绪准确率分别为 0.7812、0.7188、0.7812，平均约为 0.7604，标准差约为 0.0295，平均 group gap 约为 0.0781。也就是说，虽然 none + 0.1 在个别 seed 上能达到与 seq_gate + 0.02 相同的单次峰值，但它的波动更大、平均表现更低、group gap 也更高；相反，seq_gate + 0.02 不仅保住了高 hold-out，而且在复验中体现出更好的稳定性。

因此，阶段14最终不再保留模糊表述，而是可以明确下结论：如果项目需要一个正式的、能够代表“强化统计主干 + 条件交互 + 轻群体约束”三者有机结合的统一模型，那么应选择：

stat_hidden_channels = 96
stat_num_layers = 2
temporal_type = gru
gru_layers = 2
fusion_type = gated
gate_mode = vector
interaction_mode = seq_gate
group_loss_weight = 0.02
dropout = 0.3
standardize_sequence_input = True

若仍需保留一个更保守的稳健参照版本，则可以继续保留 interaction=none、group_loss_weight=0.02 的版本作为低 gap 对照主线；但作为阶段14的统一定稿模型，当前应明确以 seq_gate + 0.02 作为最终答案。
