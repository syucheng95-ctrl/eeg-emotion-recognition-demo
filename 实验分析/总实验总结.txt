总实验总结

本项目围绕“基于脑电数据的情绪识别”任务，已经完成了一条较为完整且逻辑清晰的实验路线。整体上，实验并不是零散地堆叠模型，而是围绕一个持续演化的核心问题不断推进：在当前比赛数据与评估协议下，真正稳定、可泛化的情绪判别信息到底位于哪个表示层级；更复杂的时序、图结构或时空模型是否能够把更细粒度的信息真正转化为跨被试泛化优势；以及在发现模型存在明显群体偏置之后，如何通过更合理的结构设计和训练约束，获得既更高分又更稳健的方案。

第一阶段的工作是建立一个可靠的传统统计基线，并完成完整的数据处理闭环。我们首先基于官方训练集，对每个 trial 进行切窗、提取微分熵（DE）特征，并在 trial 层面进行聚合，然后使用传统分类器进行被试级交叉验证与固定 hold-out 被试评估。通过对窗长、步长、频带划分、聚合方式和分类器的系统比较，最终筛选出当前最稳健的传统主线：mean+std 聚合、classic5 频带划分以及 Logistic Regression 分类器。该方案在交叉验证中的平均准确率约为 0.650，在 hold-out 上约为 0.729，同时正常组与抑郁组的性能差距仅约 0.031。这个结果说明，trial 级统计特征在当前任务中不仅有效，而且具有很强的稳健性和较好的组间平衡性，因此它不是一个简单的“弱基线”，而是整个后续实验的稳定锚点。

在此基础上，项目进一步探索了窗口级时序建模，试图回答“trial 内部的动态信息是否能带来超越传统统计的收益”这一问题。相关实验表明，窗口序列模型确实能够捕捉到额外的信息。例如，stage2_window_mean 在交叉验证中的表现已经明显高于 baseline，hold-out 也可以追平 baseline 的约 0.729。但与此同时，这类模型会出现显著的组间偏差扩大现象，典型地表现为抑郁组准确率明显更高、正常组准确率明显更低。也就是说，时序模型并不是无效，而是它虽然学到了补充信息，却更容易混入群体差异，从而削弱跨被试泛化的稳定性。这个阶段的重要意义在于，它首次提示我们：动态信息是有价值的，但不能脱离统计主干单独承担主线任务。

随后，项目系统地尝试了图结构建模路线，包括最小图模型、更强聚合方式、更强图卷积、不同图卷积核（如 GCN、Cheb、GAT）、图时序模型、自注意力图模型以及域适配 DANN 等。相关结果显示，这些图模型在交叉验证中往往并不弱，有些配置甚至能达到接近甚至高于 baseline 的 CV 表现，但在 hold-out 上始终未能稳定超过传统统计基线，通常落在约 0.625 到 0.667 区间，且组间表现也普遍不如 baseline 稳定。这一系列结果说明，当前任务中虽然 EEG 通道之间确实存在空间关系，但在现有数据量、标签粒度和先验图设计条件下，图主干并没有把空间信息稳定转化为跨被试泛化优势。项目在这一阶段实际上已经充分验证了图方向的边界：图结构不是完全无用，但不适合作为当前方案的主干。

为了进一步验证“更细粒度、更重的时空模型”是否可行，项目还构建并测试了 raw-patch A/B/C 三组模型。这些模型从比窗口级更细的时间片段出发，试图建模更局部的时空结构。然而实验结果同样显示，这条路线在当前任务上没有形成稳定优势。rawpatch_a 的 hold-out 约为 0.646，rawpatch_b 约为 0.635，rawpatch_c 更进一步下降到约 0.604，整体均明显低于 baseline。这一阶段的结论与前面的图模型实验形成了相互印证：在当前比赛数据条件下，模型越往细粒度 patch 级推进，越难稳定保住泛化性能。这也为后面“稳定信号主要位于更粗层级”这一判断提供了实证支持。

项目随后并没有直接放弃深度模型，而是进一步做了前处理消融实验，试图排除“是不是预处理限制了复杂模型性能”的可能性。具体比较了 channel z-score、local baseline correction 以及两者结合对传统基线和窗口模型的影响。实验结果非常明确：channel z-score 会显著破坏传统 DE 基线的表现，说明 trial 内幅值/能量差异本身就是任务中的有效信号；local baseline correction 对 baseline 几乎没有帮助；但这两种预处理都能一定程度上减小序列模型的组间偏置。这表明，复杂模型确实在一定程度上依赖了群体差异信息，而单靠前处理无法同时实现“更高总体准确率”和“更好组间平衡性”。这个阶段的重要意义在于，它把问题从“是否需要继续改前处理”转移到了“必须从模型结构和训练目标上处理群体混杂”。

在完成前述多条路线探索之后，项目对前面所有结果进行了统一的层级稳定性分析。这一分析表明，当前任务中最稳定、最可泛化的判别信息主要位于更粗粒度的 trial 级统计层面，而不是更细粒度的局部时空层面。传统统计基线之所以稳健，并不只是因为模型简单，而是因为它通过人工聚合主动压缩了大量局部噪声和偶然模式；而窗口模型、图模型和 raw-patch 模型则必须在更细粒度输入和更粗粒度监督之间自己寻找稳定模式，在小样本条件下更容易学到折内有效但折外不稳的结构。通过这一阶段，项目实际上完成了一个很重要的方向收束：后续研究不能再单纯依赖“把模型做得更细、更重、更复杂”来期望自然超过 baseline，而必须围绕“如何保留 coarse signal 的稳定性，同时受约束地引入动态信息”来重新设计方法。

沿着这一判断，项目进入了“融合与双分支”阶段。首先测试了两组最直接的概率融合：baseline_logreg_meanstd + stage2_window_mean，以及 baseline_logreg_meanstd + window_mlp_gru。结果表明，这两条输出层概率融合都没有带来提升，最优融合权重甚至退化为完全使用单一窗口模型，hold-out 约为 0.677，明显低于 baseline。这个结果说明，传统统计基线与序列模型之间的互补性并不能通过简单的输出平均直接利用。换句话说，动态分支中虽然可能确实有补充信息，但这种信息没有办法在后验概率层面自然合并出来。

在简单融合无效后，项目进一步提出并实现了轻量双分支模型：一条分支处理 trial 级统计特征，另一条分支处理窗口级时序特征，二者在中间进行特征级融合，再统一做情绪分类。相关实验显示，双分支路线首次真正把动态信息转化成了总体性能提升。尤其是 dual_branch_gru_concat，在 hold-out 上达到约 0.740，首次超过 baseline 的约 0.729，证明“统计分支 + 轻量时序分支”的联合建模路线是可行的。不过，这个最优版本同时暴露出明显问题：其抑郁组准确率约为 0.875，而正常组约为 0.672，组间差高达约 0.203。这说明，双分支路线虽然已经是正确方向，但如果不加额外约束，模型仍然会优先放大那些能提分却不一定足够稳健的群体相关模式。

为了解决这一问题，项目又沿着“显式建模群体因素”的思路，设计并测试了多任务双头模型。该模型仍然采用统计分支 + 序列分支的共享主干，但在融合后的共享表示上额外加入了一个群体分类头，使模型在训练时同时优化情绪分类任务和“正常/抑郁”群体分类任务。多任务实验系统比较了 concat 与 gated 两种融合结构，以及不同群体损失权重 λ 的影响。结果显示，轻度群体监督确实能够改善双分支模型的行为。其中最优配置为 multitask_gru_gated_lambda0p1：其 hold-out 情绪准确率达到约 0.750，不仅超过 baseline，也超过此前单任务双分支的最佳结果；同时其抑郁组约为 0.781，正常组约为 0.734，组间差仅约 0.047，显著优于 dual_branch_gru_concat 的约 0.203。这说明，多任务双头在当前设置下不是简单地强化群体偏置，而是通过温和地显式建模群体因素，帮助共享表示变得更有结构，从而在准确率和组间平衡性两个维度上同时取得更优折中。

进一步地，多任务实验还给出了一个很清楚的趋势：群体监督并不是越强越好。当群体损失权重 λ 从 0.1 提高到 0.3 和 0.5 时，模型的群体分类头往往变得更强，但情绪主任务并不会持续改善，反而可能出现总体准确率回落或组间偏差重新变大的情况。这一现象说明，群体信息在当前任务中并非应当被完全忽略的噪声，也不能被无限强化为主导因素；它更适合作为一种被轻度建模、被显式监控、并通过合理权重纳入训练的辅助变量。当前最优结果表明，gated 融合结构比简单 concat 更适合承载这种多任务监督，因为它更容易在统计分支和时序分支之间实现平稳权衡。

综合所有阶段的实验，可以对本项目的工作内容和研究结论做出一个较为清晰的总结。首先，项目已经完整尝试了传统统计特征、窗口级时序建模、图结构建模、更细粒度 raw-patch 时空模型、前处理消融、输出层融合、特征级双分支以及多任务群体监督等多个方向，不仅覆盖面广，而且每一条路线都有明确的问题驱动和结果对照。其次，项目的核心成果不是简单地“找到一个最高分模型”，而是系统识别出了当前比赛任务中稳定信号的主要层级，并据此逐步收束出一条最合理的方法主线。最后，从现有结果看，最有前景的方向已经非常明确：保留 trial 级统计主干，使用轻量时序分支补充动态信息，并通过轻度、多任务式的群体监督抑制模型对单一群体模式的过度依赖。

因此，若要用一句话概括本项目目前的最终方向，可以表述为：在当前比赛数据与评估协议下，最稳定、最可泛化的情绪判别信息主要来源于 trial 级统计表示；深度模型的作用不是取代这一路线，而是在其基础上以受约束的方式引入动态时序信息；当前最值得继续推进的正式方法主线，是“统计分支 + GRU 时序分支 + gated 融合 + 轻度群体多任务监督”的轻量双分支框架。这条路线既承接了前面所有实验结论，也在现阶段给出了目前最优且最平衡的综合结果。

在此基础上，项目随后进一步完成了统一模型的定稿工作，也就是把“强化统计主干”“更强双分支交互”和“轻度群体约束”三条后续路线真正收束成一个明确、可复验的正式模型，而不再只是并列摆放的几个方向。具体而言，阶段14在阶段13已有结论之上，明确将统计分支增强为 96 hidden、2 层的主干结构，保留 2 层 GRU 作为时序动态补充分支，继续使用 vector gated fusion 作为分支融合方式，并在统计分支到时序分支之间加入 seq_gate 单向条件交互，使更稳定的 trial 级统计先验先去调节时序表示；同时在共享表示上维持小权重 group loss 的轻度群体多任务监督，其中 group_loss_weight 最终确定为 0.02。换句话说，当前项目不再只是“统计主干 + 时序支路 + 群体监督”的松散组合，而是正式定型为：强化统计主干（96x2） + 2 层 GRU 时序分支 + seq_gate 单向条件交互 + vector gated fusion + 0.02 轻群体多任务监督。

这一最终定稿不是拍脑袋给出的，而是在现有 hold-out 与多 seed 复验结果基础上筛选出来的。项目将最强的统一增强版候选 seq_gate + 0.02 与对照候选 none + 0.1 做了 seed 42、52、62 的独立复验比较。结果显示，seq_gate + 0.02 三次 hold-out 情绪准确率分别为 0.7812、0.7500、0.7708，平均约为 0.7674，标准差约为 0.0130，平均 group gap 约为 0.0677；而 none + 0.1 三次 hold-out 为 0.7812、0.7188、0.7812，平均约为 0.7604，标准差约为 0.0295，平均 group gap 约为 0.0781。也就是说，虽然 none + 0.1 在个别 seed 上也能冲到较高 hold-out，但其波动更大、平均表现更低、组间差也更高；相较之下，seq_gate + 0.02 在保持高 hold-out 的同时，体现出了更好的整体稳定性。因此，若要给出当前项目最明确、最适合作为正式方法主体的答案，那么就应选择 seq_gate + 0.02 这一版本；若要保留一个更保守的稳健参照，则仍可以继续保留 interaction=none、group_loss_weight=0.02 的版本作为对照主线。
